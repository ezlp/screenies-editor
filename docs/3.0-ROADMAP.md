# ScreeniesEditor 3.0 — Roadmap

> **3.0 = "The Theme Update."** Turn the whole interface into something the user
> can dress — a real theme engine — then restructure the editor, shell, and
> gallery around it, without adding weight.
> Interactive concept: <https://ezlp.github.io/screenies-editor/>

This is an engineering plan, not just a feature list. Code sketches use egui
0.29 APIs; exact field names should be confirmed against the 0.29 docs at
implementation time (the dev sandbox has no `cargo`, so nothing here is
compile-checked — CI does that on push).

---

## 1. The technical situation today

**Workspace.** Two crates:
- `core/` (`screenies-core`) — shell-independent engine: `render` (layout, filters, compose), `fonts` (fontdb + ab_glyph, for the *rendered image* only), `gallery` (folder listing), `chatlog_library`. Unit-tested. **No egui.**
- `native/` (`screenies-native`) — the eframe/egui **0.29** shell. Excluded from the workspace; built with `cd native && cargo run`. ~2.4k LOC across `main.rs` (shell), `editor.rs` (1568 lines, ~90% of the UX), `gallery.rs`, `chatlog_browser.rs`, `i18n.rs`.

**Shell & state** (`main.rs`):
- `enum Screen { Menu, Editor, Gallery, Settings }` (`main.rs:42`) drives a `CentralPanel` match (`:166`). Navigation is hub-and-spoke: a `TopBottomPanel::top("nav")` (`:146`) with a `← Menu` button appears on every non-Menu screen. The landing `menu()` (`:206`) is a `vertical_centered` column of `add_sized([440,62], Button)` tiles (`menu_tile`, `:268`).
- `struct App { screen, editor: EditorState, gallery: GalleryState, dark: bool, lang, ui_scale }`.
- Persistence: `struct Settings` (`:74`) is serde-serialized under key `"settings"` via eframe storage. Fields: `dark, font, lang, text_size, line_gap, filters: FilterValues, ui_scale, chatlog_folder`. Loaded in `App::new` (`:115`), written in `App::save` (`:189`) — editor prefs are pulled back out via `editor.prefs()`.

**Theming today** is two lines, run every frame (`main.rs:133`):
```rust
ctx.set_visuals(if self.dark { egui::Visuals::dark() } else { egui::Visuals::light() });
ctx.set_zoom_factor(self.ui_scale);
```
Stock egui presets. **No custom `Style`, no accent, no rounding/spacing of our own, no custom UI fonts.** The only brand color, lavender `#c2a2da`, is a magic number in the crop painter (`editor.rs:885`), not a token. Other inline literals: error red `Color32::from_rgb(220,90,90)` ×4 (`editor.rs:609`, `gallery.rs:51`/`:183`, `chatlog_browser.rs:111`), censor-selection blue/grey (`editor.rs:772`), sticker-selection amber (`:827`).

**Editor** (`editor.rs`): `SidePanel::left("controls")` (`:271`) wraps **one `ScrollArea::vertical`** with ~14 stacked groups (`controls()`, `:298–611`); `CentralPanel` holds the hand-painted `preview()` (`:719`) / `preview_crop()` (`:845`). State is `EditorState` with a `Vec<Doc>` (per-photo) + global text/filter prefs.

**Gallery** (`gallery.rs`): `GalleryState` lazily decodes thumbnails (`THUMBS_PER_FRAME=6`) into a `HashMap` cache, lays them out in a `horizontal_wrapped` grid of `150×112` `ImageButton`s (`thumb_cell`, `:100`); filename is a **hover tooltip only** (`:111`); the decoding placeholder is a bare `"…"` (`:109`). Clicking opens a floating `egui::Window` preview (`preview_window`, `:152`). **The gallery folder is not persisted** (unlike `chatlog_folder`), so it's empty every launch.

**Constraints.** No webview; immediate-mode (UI rebuilt every frame → theme swaps are free); target old laptops; dev sandbox can't build (author compiles, CI `native-check` gates). Git push over SSH works; `main` is directly pushable; releases cut from tags via `native-preview.yml`.

## 2. Guiding constraints

- **Never a downgrade.** Every milestone leaves a runnable, non-regressed app (strangler-fig). No milestone removes a feature; B/C/D *restructure*.
- **Featherweight.** No webview, no heavy assets. The bundled icon-font subset (C) is the only new asset, a few KB.
- **Colors flow through `ui.visuals()`.** Screens read theme colors from egui's `Visuals` instead of literals, so B/C/D inherit theming for free once A lands.
- **Verifiable.** `build.yml`'s `native-check` (`cargo check` on `native/`) gates every push; behavior testing is manual.

## 3. Architecture decisions

**D1 — Theme is data written into egui `Visuals`, not a palette threaded everywhere.**
`apply()` maps a `Theme` onto `Visuals` + `Style` and calls `ctx.set_visuals` / `ctx.set_style`. Screens then read `ui.visuals().error_fg_color`, `.selection.stroke`, `.hyperlink_color`, etc. This means the color sweep (A) is a *read-site* change, not new plumbing. Two semantic colors have **no** `Visuals` slot — `ink_dim` (caption/weak text) and the distinct censor-vs-sticker selection hues. For those we keep the resolved `Theme` on `App` and pass `&Theme` into the ~4 screen `ui()` calls (explicit, testable, low churn).

**D2 — `Theme` lives in `native/` (UI-only), never `core/`.** It depends on `egui::Color32`; `core` stays egui-free.

**D3 — Settings schema evolves additively with migration.** `#[serde(default)]` on the struct means new fields are safe. Keep `dark: bool` one version for migration → `theme` id; keep the door open for a future `#[serde(rename)]`/version tag if the shape changes more.

**D4 — egui 0.29 API notes.** Rounding is `egui::Rounding` (renamed `CornerRadius` in ≥0.31 — do not use that name here). `Rounding::same(f32)`. `WidgetVisuals { bg_fill, weak_bg_fill, bg_stroke, rounding, fg_stroke, expansion }` across `widgets.{noninteractive,inactive,hovered,active,open}`. `Selection { bg_fill, stroke }`. Semantic slots: `error_fg_color`, `warn_fg_color`, `hyperlink_color`, `panel_fill`, `window_fill`, `faint_bg_color`, `extreme_bg_color`, `window_stroke`. Density via `Style.spacing.{item_spacing, button_padding, interact_size}`.

**D5 — Icons: bundled icon-font subset over hand-painted shapes.** A subset TTF (Lucide/Tabler/Phosphor) loaded through `FontDefinitions` gives ~30 crisp glyphs that inherit color/size and the accent, for a few KB — cheaper in code than painting each. Alternative (pure `Painter` shapes) stays asset-free but balloons code; rejected for the icon *set*, kept for one-offs.

## 4. Milestones

### Milestone A — Theme engine · ships `v3.0.0-alpha`

The headline; self-contained; **no layout change**; de-risks B/C/D.

**A1 — `native/src/theme.rs` (new).**
```rust
use eframe::egui::{self, Color32, Rounding, Stroke, Visuals};

#[derive(Clone)]
pub struct Theme {
    pub id: &'static str, pub name: &'static str, pub dark: bool,
    pub bg: Color32,         // panel_fill      — central + side panels
    pub surface: Color32,    // window_fill / faint_bg — cards, popups, headers
    pub surface_2: Color32,  // extreme_bg      — text edits / inset fields
    pub ink: Color32,        // override_text_color
    pub ink_dim: Color32,    // weak text (no Visuals slot → passed via &Theme)
    pub accent: Color32,     // selection / active widget / hyperlink
    pub accent_ink: Color32, // text on accent fills (primary buttons)
    pub border: Color32,     // widget + window strokes
    pub error: Color32, pub warn: Color32, pub good: Color32,
    pub rounding: f32, pub dense: bool,
}

pub fn builtins() -> &'static [Theme] { &BUILTINS } // 7 consts
pub fn by_id(id: &str) -> &'static Theme {
    builtins().iter().find(|t| t.id == id).unwrap_or(&builtins()[0])
}

impl Theme {
    pub fn apply(&self, ctx: &egui::Context, accent: Option<Color32>, scale: f32) {
        let accent = accent.unwrap_or(self.accent);
        let mut v = if self.dark { Visuals::dark() } else { Visuals::light() };
        v.panel_fill = self.bg;
        v.window_fill = self.surface;
        v.faint_bg_color = self.surface;
        v.extreme_bg_color = self.surface_2;
        v.override_text_color = Some(self.ink);
        v.hyperlink_color = accent;
        v.error_fg_color = self.error;
        v.warn_fg_color = self.warn;
        v.window_stroke = Stroke::new(1.0, self.border);
        v.selection.bg_fill = accent.linear_multiply(0.35);
        v.selection.stroke = Stroke::new(1.0, accent);

        let r = Rounding::same(self.rounding);
        for w in [&mut v.widgets.noninteractive, &mut v.widgets.inactive,
                  &mut v.widgets.hovered, &mut v.widgets.active, &mut v.widgets.open] {
            w.rounding = r;
            w.bg_stroke = Stroke::new(1.0, self.border);
            w.fg_stroke = Stroke::new(1.0, self.ink);
        }
        v.widgets.inactive.weak_bg_fill = self.surface_2; // idle buttons
        v.widgets.active.bg_fill = accent;                // pressed / primary
        v.widgets.active.fg_stroke = Stroke::new(1.0, self.accent_ink);
        ctx.set_visuals(v);

        let mut style = (*ctx.style()).clone();
        let sp = &mut style.spacing;
        let (gap, pad, h) = if self.dense { (4.0, 3.0, 22.0) } else { (8.0, 6.0, 26.0) };
        sp.item_spacing = egui::vec2(if self.dense {6.0} else {8.0}, gap);
        sp.button_padding = egui::vec2(if self.dense {6.0} else {10.0}, pad);
        sp.interact_size.y = h;
        ctx.set_style(style);

        ctx.set_zoom_factor(scale); // replaces main.rs:140
    }
}
```

- [ ] `apply()` as above; unit-test that each builtin produces sane contrast (ink vs bg luminance ratio ≥ 4.5).

**A2 — `main.rs` wiring.**

Lowest-risk, rides on A's tokens. All in `gallery.rs` (+ maybe `core::gallery::Item`).

- [ ] **Toolbar.** folder + count · `TextEdit` search (filter `items` by name) · sort combo (Newest via fs mtime / Name / Size) · size segmented (S/M/L → swap `THUMB_W/H` presets, currently the consts at `:14`).
- [ ] **Captions.** `thumb_cell` (`:100`) renders a caption *under* the image — `name · {w}×{h} · {mtime}` — instead of the hover tooltip (`:111`). Needs dims (from the decoded thumb's source size) + mtime (`std::fs::metadata`); extend `Item` or fetch lazily beside the thumbnail.
- [ ] **Skeleton.** Replace the `"…"` placeholder (`:109`) with a themed shimmer: a `painter.rect_filled` in `faint_bg_color` with a time-driven highlight (`ui.ctx().request_repaint()` while any thumb is still decoding — the grid already repaints during decode at `:90`).
- [ ] **Lightbox.** Replace the floating `Window` (`preview_window`, `:152`) with an in-app overlay over the central area: dim scrim (`Area` + `painter.rect_filled` at `Color32::from_black_alpha`), the preview centered, a metadata caption + action row (Open in editor / Reveal in folder / Copy path), and **←/→** navigation: `if ui.input(|i| i.key_pressed(Key::ArrowRight)) { self.selected = next }`. ## 5. Cross-cutting

- **Persistence schema** grows: `theme, accent, dense, gallery_folder` (all `#[serde(default)]`). One migration path (`dark` → `theme`). Editing-defaults (font/size/gap/filters) are already persisted — Settings just gains UI to edit them.
- **i18n:** each milestone adds keys to `i18n.rs`; several 2.x strings bypass i18n (chatlog popup, photo-tab labels, error strings) — fold those in opportunistically.
- **Testing reality:** no `cargo`/GUI here → CI `native-check` is the compile gate; behavior + visual checks are manual by the author. Add `core`/`theme` unit tests where logic is pure (theme contrast, settings migration, gallery sort/filter).

## Out of scope for 3.0 (deferred)

- **3.1 — Make your own:** live custom-theme editor (tweak any token), import/export a theme as a small `.toml`, persist last N custom themes.
- **3.2 — Share the look:** drop a `.toml` in a themes folder → it appears in the gallery; community theme packs; optional per-screenshot theme presets.
