# ScreeniesEditor 3.0 — Roadmap

> **3.0 = "The Theme Update."** Turn the whole interface into something the user
> can dress — a real theme engine — and restructure the editor and navigation
> around it, without adding weight. Interactive concept: <https://ezlp.github.io/screenies-editor/>

## Why themes, and why now

In a native egui app a theme is a `Style`/`Visuals` struct — a dozen colors, not
a stylesheet. So rich theming is the highest value-per-byte feature this app can
ship: maximum personality, ~zero bytes, no new assets, no repaint cost (the UI is
rebuilt every frame anyway). The 1.x Tauri build had a CSS-variable theme system
that the native rewrite dropped to `Visuals::dark()` / `Visuals::light()`; 3.0
brings that capability back as plain Rust data.

## Guiding constraints

- **Never a downgrade.** Every milestone leaves a fully runnable, non-regressed
  app (strangler-fig). No milestone removes a feature; B and C only *restructure*.
- **Featherweight.** No webview, no heavy assets. A bundled icon-font subset (C)
  is the only new asset and stays a few KB.
- **egui 0.29, immediate mode.** Colors flow through `ui.visuals()` so screens
  read theme colors instead of hardcoded literals.
- **Verifiable in CI.** `build.yml`'s `native-check` (`cargo check` on `native/`)
  gates every push; runtime testing is manual (no GUI/cargo in the dev sandbox).

## Milestone A — Theme engine  ·  ships `v3.0.0-alpha`

The headline. Self-contained, lowest risk (no layout change), de-risks B and C
because once colors flow through `visuals`, both inherit theming for free.

**New module `native/src/theme.rs`**
- [ ] `struct Theme { id, name, dark, bg, surface, surface_2, ink, ink_dim, accent, accent_ink, border, error, warn, good, rounding: f32, dense: bool }` (colors = `egui::Color32`).
- [ ] `pub fn builtins() -> &'static [Theme]` — 7 themes: Midnight (default, honors the `#c2a2da` lavender), Paper, Slate, Ember, San Andreas, Frost, Contrast (a11y).
- [ ] `impl Theme { fn apply(&self, ctx, accent_override: Option<Color32>, scale: f32) }` — build `Visuals`, set `panel_fill`/`window_fill`/`override_text_color`/`hyperlink_color`/`selection`/`widgets.*.bg_fill`/`widgets.*.rounding`/`error_fg_color`/`warn_fg_color`, then `Style.spacing` for density.

**Wire-up `native/src/main.rs`**
- [ ] Replace `set_visuals(if dark {..} else {..})` (main.rs:133) with `theme.apply(ctx, accent, ui_scale)`.
- [ ] `Settings`: add `theme: String`, `accent: Option<[u8;3]>`, `dense: bool`; migrate legacy `dark: bool` → `"midnight"`/`"paper"` on load.
- [ ] Rebuild the near-empty Settings screen into the personalization home: theme swatch gallery + accent picker + density toggle (keep UI-scale + language).

**Color-literal sweep** (replace magic numbers with theme reads)
- [ ] error red `Color32::from_rgb(220,90,90)` ×4 (editor.rs:609, gallery.rs:51/183, chatlog_browser.rs:111) → `ui.visuals().error_fg_color`.
- [ ] crop accent `(194,162,218)` (editor.rs:885) → `visuals.selection.stroke`.
- [ ] censor/sticker selection colors (editor.rs:772–827) → accent + neutral.

**Acceptance:** 7 themes switch live and persist across launches; accent override
applies; density changes spacing; no hardcoded error/selection colors remain;
`cargo check` green; **no layout change** vs 2.1.

## Milestone B — Editor you operate, not scroll  ·  ships `v3.0.0-beta`

Restructure the single 14-group sidebar scroll (editor.rs:298). All in `editor.rs`;
a re-grouping, not a feature change. Higher churn → verify carefully.

- [ ] `enum Tool { Photo, Crop, Chatlog, Text, Fx }` + `active_tool` on `EditorState`.
- [ ] Segmented tool switcher at the top of the controls panel; split `controls()` into `tool_photo/tool_crop/tool_chatlog/tool_text/tool_fx` — only the active tool's controls render.
- [ ] Bottom action bar (`TopBottomPanel::bottom` inside the editor): **Export** as accent-filled primary + Save draft; remove Export from the scroll (editor.rs:604).
- [ ] Floating preview toolbar (`Area` anchored over the central panel): undo, redo, zoom, fit.
- [ ] Keep tab-scoped vs global control semantics intact (`Doc` per-photo state).

**Acceptance:** every 2.x control reachable via a tool tab; Export always visible;
undo/redo/zoom float over the preview; nothing lost; `cargo check` green; manual
smoke test of each tool + export.

## Milestone C — Navigation & icons  ·  ships `v3.0.0` (STABLE)

- [ ] Persistent `SidePanel::left` nav rail (Editor / Gallery / Settings) — replace the hub-and-spoke menu bounce (main.rs:146–171).
- [ ] Remove the theme/lang toggles duplicated in nav bar + menu + settings; single home in Settings.
- [ ] Coherent monochrome icon set replacing the 18 default-font emoji — a small bundled icon-font subset via `FontDefinitions` (inherits the accent).
- [ ] Bump **both** `version =` lines in `native/Cargo.toml` (`[package]` + `[package.metadata.generate-rpm]`) to `3.0.0`.

**Acceptance:** jump between screens without returning to a menu; one place for
theme/lang/personalization; consistent icons; `cargo check` green; clean `v3.0.0`
tag builds green installers via `native-preview.yml`.

## Out of scope for 3.0 (deferred)

- **3.1 — Make your own:** live custom-theme editor (tweak any token), import/export a theme as a small `.toml`, persist last N custom themes.
- **3.2 — Share the look:** drop a `.toml` in a themes folder → it appears in the gallery; community theme packs; optional per-screenshot theme presets.

## Release & verification process

- Branch per milestone off `main` → push over SSH → CI `native-check` (`cargo check`) gives green/red via the public Actions API → PR link → **you** run/test locally.
- Tags: a `-` in the tag ⇒ pre-release (`v3.0.0-alpha`, `-beta`); a clean tag (`v3.0.0`) ⇒ stable. Tag push builds `.exe`/`.deb`/`.rpm`/AppImage via `native-preview.yml`.
- Nothing touches stable until Milestone C.

## Summary

| Milestone | Focus | Risk | Ships as | Home |
|---|---|---|---|---|
| **A** | Theme engine + 7 themes + accent/density + Settings home | Low | `v3.0.0-alpha` | new `theme.rs`, `main.rs` |
| **B** | Editor tool-rail + action bar + floating toolbar | Medium | `v3.0.0-beta` | `editor.rs` |
| **C** | Nav rail + unified settings + icon set | Medium | `v3.0.0` stable | `main.rs`, assets |

**Definition of done for 3.0:** all three milestones merged; a user can pick from
7 themes (+ custom accent + density), operate the editor via the tool rail with a
persistent Export, and move between screens from a nav rail — on the same
featherweight native binary. Start with **A**.
