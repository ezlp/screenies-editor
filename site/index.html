Where 3.0 starts

An honest look at the app today

The engine is solid and the feature set is real. What's thin is the surface — the parts a first-time user actually touches. Three things 3.0 inherits:

Theming

Two hard-coded looks

The entire theme system is one line run every frame: set_visuals(dark ? dark() : light()). There's no accent, no rounding, no spacing of your own. The one brand color — that lavender #c2a2da — is a magic number buried in the crop painter, not a token.

The editor

One sidebar, 14 groups, one scroll

Undo, photo tabs, load, crop, cinematic, chatlog, palette, anchor, text, filters, censor, stickers — all stacked in a single 320 px column. Only 4 collapse. Export, the main action, sits at the very bottom.

Navigation

Back to the menu, every time

It's hub-and-spoke: to go Editor → Gallery you bounce through the landing menu. And the theme + language toggles are duplicated in three places — nav bar, menu, and a near-empty Settings screen.

Pillar one

A theme is data, not CSS

Define a theme as a dozen values — grounds, ink, one accent, corner radius, density. egui rebuilds the UI from it on the next frame. Ship a handful, let people build their own.

…and it compiles to about this much Rust

// A theme is plain data. No assets, no I/O, no allocation.
struct Theme {
    name:    &'static str,
    dark:    bool,
    bg:      Color32,   // window / panel ground
    surface: Color32,   // sidebars, cards, fields
    ink:     Color32,
    accent:  Color32,   // the one that carries the identity
    rounding: f32,
    dense:   bool,
}

impl Theme {
    fn apply(&self, ctx: &egui::Context) {
        let mut v = if self.dark { Visuals::dark() } else { Visuals::light() };
        v.panel_fill        = self.bg;
        v.window_fill       = self.surface;
        v.override_text_color = Some(self.ink);
        v.selection.bg_fill = self.accent.gamma_multiply(0.35);
        v.widgets.active.bg_fill = self.accent;   // primary buttons
        v.hyperlink_color   = self.accent;
        ctx.set_visuals(v);

        let mut s = (*ctx.style()).clone();
        s.spacing.item_spacing = if self.dense { vec2(6.0, 4.0) } else { vec2(8.0, 8.0) };
        ctx.set_style(s);
    }
}

Swapping themes is a struct assignment. 3.1 makes Theme serde-serializable → save, share, and drop .toml theme files into a folder to see them in the gallery.

Pillar three

Navigation that remembers where you are

Nav rail

No more round-trips

A slim persistent rail — Editor, Gallery, Settings — always down the left edge. Jump straight between screens instead of returning to a landing menu each time.

Settings

One home for personalization

Themes, accent, density, UI scale, language, default chatlog font — the things scattered across the nav bar, menu, and a near-empty Settings screen today, gathered into one real place.

Icons

One coherent set

The 18 default-font emoji (📂 🗑 🎬 ✏ …) that render inconsistently give way to a single monochrome line-icon set that inherits the theme accent — like every icon on this page.

The constraint

Featherweight by construction

The whole point of the native rewrite was a small, fast app for old laptops with no webview. Themes don't threaten that — they're the one big feature that costs almost nothing.

Why it's basically free

In an Electron or webview app, theming means shipping stylesheets and asking a browser engine to recompute layout. In egui there's no DOM to reflow — the UI is rebuilt every frame from your Style, so a theme swap just hands it different numbers. Maximum personality, near-zero cost. That's precisely why themes are the right flagship for a lightweight app.

It's a homecoming, not a new cost

The old 1.x Tauri build actually had a CSS-variable theme system — the migration notes even say "custom themes can be added later." The native rewrite dropped it to defaults to ship. 3.0 brings that capability back, this time as plain Rust data that weighs nothing.

Sequencing

How it could ship

Themes first because they're the headline and the cheapest big win. The IA work rides along since the accent system touches every screen anyway.
